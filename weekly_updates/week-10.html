LLM-Based Time Series Forecasting

Weekly Progress Update – Applying LLM-style modeling to solar time-series data.

1) Objective

  • Use time-series solar magnetic parameters as input to an LLM-style architecture.
  • Evaluate different multi-step forecasting strategies.

2) Data Representation & Windowing

Input Configuration

  • Sequence length: 24 time steps
  • Sliding window shift: 1
  • Prediction horizon: 6 steps ahead
  • Features used: R_VALUE, TOTUSJH

Windowing Strategy

  • Each sample consists of 24 past timestamps.
  • The window moves forward by one time step.
  • The model predicts the next 6 future values.

3) Architectures Implemented

A) Direct Multi-Step Forecasting

  • Model predicts all 6 future steps in a single forward pass.
  • No recursive feedback from predicted outputs.

B) Autoregressive Forecasting

  • Predict one step at a time.
  • Each predicted output is fed back as input for the next step.

C) Autoregressive + RevIN

  • Applies reversible instance normalization (RevIN).
  • Normalizes input before prediction and restores original scale afterward.

4) Limitations

  • Class imbalance was not addressed.
  • Only two physical features were used.
  • No feature expansion or multimodal inputs yet.

8) Key Takeaways

  • Direct forecasting significantly outperforms autoregressive in this setup.
  • Error accumulation likely affects autoregressive performance.
  • RevIN requires deeper investigation to interpret scaling behavior.
  • Next step: handle imbalance, add more features, test stronger architectures.
